# 信源命中效率面板操作说明

## 面板回答什么问题

这个面板衡量的是“抓到的内容有多少适合本站的动漫 / IP / 文创选题”，不是媒体事实可信度评分。

- 低分率 = 相关性评分 0–3 的文章数 / LLM 已评分文章数。
- 有效命中率 = 相关性评分 7–10 的文章数 / LLM 已评分文章数。
- 抓取发现、去重入库、LLM 已评分、高分、最终精选组成完整处理漏斗。
- 至少有 20 条已评分样本才显示“正常、需观察或重点复核”的结论；不足 20 条只标记“样本不足”。

## 数据口径

抓取任务和 LLM 任务会把每次的来源统计、评分结果和少量样本写入 `cron_logs.details`。审计先于低分文章清理，因此低分文章从 `articles` 删除后仍会计入分母和低分数。

部署前的数据只能从现存文章估算。界面会标记“历史估算”，因为旧流程删除过低分文章，这部分数据可能偏乐观；新任务运行后会自动切换到完整审计口径，不需要数据库迁移。

## 建议操作流程

1. 先看样本量。少于 20 条时继续观察，不停用。
2. 展开来源，检查低分样本是否确实与本站定位无关，同时对照高分样本。
3. 查看 7 天与 30 天：短期异常但 30 天正常时先观察；两个周期持续偏高再调整。
4. 如果内容有价值但杂讯多，先修改栏目 URL 或抓取规则。
5. 若命中率长期偏低，先“降低频率”；降频来源只参加一半定时轮次。
6. 连续两个周期仍然偏低且低分样本确认无价值，再“停用来源”。
7. 恢复停用来源前，必须在信息源管理页测试成功。

“继续观察、降低频率、停用来源”都是人工决策。系统不会只凭低分率自动停用信息源。

## 状态阈值

| 状态 | 条件 | 建议 |
| --- | --- | --- |
| 样本不足 | 已评分少于 20 条 | 继续积累样本 |
| 正常 | 低分率低于 30% | 维持当前策略 |
| 需观察 | 低分率 30%–59% | 检查栏目和样本，必要时降频 |
| 重点复核 | 低分率不低于 60% | 连续两个周期异常后再考虑停用 |
